### 📝 README 追記用：アップデート内容まとめ（v2.4）

#### ✅ 実装できたこと（新機能と最適化）
1. **共役熱伝達（Conjugate Heat Transfer）のネイティブ対応**
   - 固体セル内部でも温度の分布関数 $g$ を解くようにアーキテクチャを変更。
   - 流体-固体界面での特別な境界処理（ディリクレ境界など）を廃止し、物質ごとの熱伝導率（$\tau_g$）の違いによる自然な熱移動・熱抵抗のシミュレーションを実現。
2. **境界条件の完全オブジェクト指向化（Strategyパターン）**
   - `VelocityInlet`, `PressureOutlet`, `IsothermalWall`, `ConstantHeatFluxWall` などの境界条件を独立したクラスに分離。
   - `config.py` のディクショナリ設定（IDベース）から、パズルのように自由に境界条件を組み合わせて適用できる `BoundaryManager` を実装。
3. **物理モデルのプラグイン化（疎結合アーキテクチャ）**
   - 多相流（Shan-Chenモデル）や自然対流（ブシネスク近似浮力）などを LBM ソルバ本体から切り離し、`PhysicsManager` として独立。
   - 将来的な「相変化モデル（沸騰・潜熱）」をプラグインとして追加しやすい基盤を確立。
4. **VRAM使用量の大幅削減（インプレース移流の擬似実装）**
   - 流体と熱の巨大な一時バッファ配列（`f_post`, `g_post`）を廃止。
   - 衝突（Collide）と移流（Stream）を1つの統合カーネル `collide_and_stream` にまとめ、計算結果を直接 `_new` 配列にPushすることで、VRAM使用量を従来の約60%にまで圧縮し、高解像度化（128x128x512等）に貢献。
5. **LBMパラメータ設計・安定性判定ツールの構築**
   - 物理パラメータ（Re数、Pr数）とセル基準の無次元数（セルRe数、セルPe数）から、LBMの数学的安定条件（$\tau > 0.5$）を満たしているかを実行前に判定するツールを実装。

#### ❌ 実装できなかったこと（技術的限界と今後の課題）
1. **FP16（半精度浮動小数点数）によるメモリ節約**
   - **理由：** LBMの分布関数は `1.0` を基準とした $10^{-5}$ オーダーの極めて微小な変動（非平衡成分）で物理現象を表現する。FP16では精度の幅が足りず、丸め誤差によって非平衡成分が消滅し、数ステップで計算が完全に破綻（爆散）するため採用不可。
2. **Taichi の SNode（スパースデータ構造）によるメモリ削減**
   - **理由：** Taichi の `pointer` 構造は、`ti.ndrange` 等で全空間を走査するループを回すと、アクセスした全座標にメモリブロックを強制割り当てしてしまう仕様がある。LBMのような全セル走査型アルゴリズムと極めて相性が悪く、逆に管理オーバーヘッドでVRAMパンクを引き起こしたため撤回（Dense構造に回帰）。
3. **速度場と温度場の分離解法（交互計算）によるメモリ節約**
   - **理由：** 相変化（沸騰・キャビテーション）のような強連成かつ極めてタイムスケールの短い非定常現象を捉えるためには、速度・温度・密度を同一タイムステップで同時に解き進める必要があり、物理的要請により不採用。
