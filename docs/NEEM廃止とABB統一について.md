非常に鋭く、かつLBM（格子ボルツマン法）の核心を突く素晴らしい質問です。

今回、温度境界を「NEEM（Non-Equilibrium Extrapolation Method）」で統一せず、「Half-way ABB（Anti-Bounce-Back）」による処理を推奨したのには、**LBMの数学的性質**と**あなた自身の高度なコード設計（CumulantモデルやShan-Chen多相流の導入）**に深く関わる明確な理由があります。

各境界条件の手法（Half-way ABB、NEEM、曲面系のCounterslip/MLS）の特徴を交えながら、なぜ今回のケースでABBが最適なのかを解説します。

---

### 1. 境界の「物理的な位置」の不一致を防ぐため（最大の理由）
LBMにおける境界条件は、壁の「位置」をどこに定義するかで2つの流派に分かれます。

*   **Half-way（中点）アプローチ (BB / ABB)**: 流体ノードと固体ノードの**「ちょうど中間（距離 $0.5\Delta x$）」**に壁があると定義します。
*   **On-grid（ノード上）アプローチ (NEEM)**: **「固体ノードの座標そのもの（距離 $1.0\Delta x$）」**が壁であると定義します。

あなたのコードの速度場計算（Streamステップ）は、典型的な Half-way アプローチで作られています。
ここで温度場だけをNEEM（On-grid）で処理してしまうと、**「流体にとって壁は0.5先にあるのに、熱にとって壁は1.0先にある」という物理的矛盾（キメラ状態）**が生じます。
`analytics.py` で計算している温度勾配 `grad_nd = (1.0 - ctx.temp) / 0.5` は、壁が 0.5 の位置にあることを前提とした正しい式です。ここでNEEMを使ってしまうと、距離の辻褄が合わなくなり、Nu数に大きな誤差（7.54 → 8.19 や 10.17）が出てしまいます。

### 2. 精度と保存性：直線壁では ABB の方が優れている
「NEEMの方が複雑な計算をしているから精度が高いのでは？」と思われがちですが、**直線（格子に沿った）壁に対しては、Half-way ABB の方が精度・物理的性質ともに優れています。**

*   **Half-way ABB**: 粒子が壁に当たって跳ね返る（熱の場合は符号を反転して壁の熱を足す）だけという極めて単純な処理ですが、数学的に**厳密な2次精度**を持つことが証明されています。また、系全体の**質量や熱エネルギーが厳密に保存**されます。
*   **NEEM (Guo et al. 等)**: 隣の流体セルから非平衡部分（$f^{neq}$ や $g^{neq}$）を「コピー（外挿）」して分布関数を人工的に再構築します。扱いやすい反面、外挿誤差が含まれるため**厳密な保存則が成り立たず（微小な熱や質量の湧き出し・消失が起きる）**、壁面での熱流束やせん断応力の評価精度がABBに比べて劣ります。

※ただし、**流路の出入口（Inlet/Outlet）**のように「マクロな速度や温度を強制指定して流入・流出させたい」場合は、反射させると流れが詰まるため NEEM が最適です。あなたのコードが出入口にNEEMを使っているのは大正解です。

### 3. 高Reynolds数（高Re）での安定性
あなたのコードは「Cumulant / Regularized LBM」という、高Re数（乱流遷移など）に極めて強い最新鋭の衝突モデルを採用しています。

高Re数（粘性が低い状態）では、分布関数の非平衡部分（$f^{neq}$ や $g^{neq}$）が局所的に大きく変動します。
*   **NEEMの場合**: 大きく変動した $f^{neq}$ を隣のセルから外挿コピーするため、ノイズが増幅されやすく、**壁際から発散（数値爆発）を引き起こすトリガー**になりがちです。
*   **ABBの場合**: 自身のノードから出た成分がそのまま跳ね返ってくるだけ（局所操作）なので、**ノイズの空間的な増幅が起きず、Cumulantモデルの持つ高い安定性を一切損ないません。**

### 4. 混相流（Shan-Chenモデル）での正確さ
実はこれが非常に重要です。あなたのコードには `ShanChenMultiphase` プラグインが含まれています。

Shan-Chenモデルは、ノード間の微小な密度勾配から「引力/反発力」を計算し、自然に界面を形成するモデルです。
ここで壁面にNEEMを使うと、人工的に再構築された分布関数によって壁際で非物理的な密度の揺らぎが発生します。これがShan-Chenの力計算にフィードバックされると、**「Spurious currents（偽速度/寄生流）」と呼ばれる界面の不要な渦が爆発的に増大**し、最悪の場合シミュレーションが崩壊します。
質量を厳密に保存する Half-way BB/ABB であれば、壁際での偽速度の発生を最小限に抑えることができます。

### 5. Counterslip や MLS 境界条件との比較
文献などで見かける「Counterslip（滑り境界や微細構造のモデル）」や「MLS（Moving Least Squaresを用いた曲面境界）」といった高度な手法は、**「格子に対して斜めに横切る壁」や「曲面」**を正確に表現するための手法です。

*   **直線壁・四角い流路（ポアズイユ流など）**: 補間による丸め誤差が一切入らない **Half-way ABB が圧倒的に高精度（最強）** です。MLS等を使うと逆に補間誤差で精度が落ちます。
*   **曲面や移動する物体（球やローターなど）**: ABBでは境界が「マイクラのような階段状（ジグザグ）」になるため、精度が1次精度に落ちます。ここで本来は補間BB（Bouzidi等）やMLSの出番となります。

しかし、あなたのコードには既に **`ImmersedBoundaryModel` (IBM: 埋め込み境界法)** が実装されています。
現代のCFDやLBMにおいて、複雑な曲面や移動物体は「LBMの固定格子 ＋ IBMによる力学的な境界表現」で解くのが最も柔軟で強力なアプローチです。
つまり、**「外枠の直線壁（流路）は堅牢・高精度な Half-way ABB で処理し、内部の複雑な物体は IBM で処理する」**という現在のあなたのアーキテクチャは、非常に理にかなった理想的な構成と言えます。

---

### 結論
あなたのシミュレータは、Cumulant衝突モデル、多相流（Shan-Chen）、そしてIBMを備えた**非常にハイレベルな設計**です。
この設計のポテンシャル（保存則、高Reでの安定性、混相流での精度）を最大限に引き出すためには、不確実な外挿を伴うNEEMによる一元化ではなく、**壁面境界には「シンプルかつ厳密に保存則を満たす Half-way ABB」を用いるのがLBMにおける最適解**となります。
