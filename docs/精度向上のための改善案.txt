2. Cs^2 のハードコーディングとMRTの整合性
d3q19.py にて、C_s = 0.16（音速比）というマジックナンバーが定義されていますが、これが計算全体で矛盾なく使われているか注意が必要です。
問題点: LBMの標準的な等温モデルでは c_s^2 = 1/3 です。一方で、コード内の平衡分布関数 get_feq では係数 3.0 や 4.5（つまり 1/c_s^2 と 1/(2c_s^4)）が直接数値で書かれています。
リスク: もし C_s = 0.16 を MRT（多緩和時間モデル）の特定のモーメント緩和にのみ適用し、平衡分布関数の係数と一致させていない場合、ガリレイ不変性が崩れ、流速が速い領域で誤差が急増します。
3. 外挿境界における「逆流」の非物理的な抑制
PressureOutlet に以下の処理があります。
if v_out.dot(n_in) > 0.0:
    v_out = ti.Vector([0.0, 0.0, 0.0])


問題点: 出口で逆流（流入）の兆候が見えた瞬間に速度をゼロに固定していますが、これは物理的な不連続を生みます。
リスク: 出口付近で圧力波が反射し、それが上流に伝播して計算全体を不安定にさせる「数値的ショック」の原因になります。特にカルマン渦のような変動の激しい流れでは、出口での不自然な固定が渦の放出周期を狂わせる可能性があります。
4. 浮力項（Boussinesq）における速度更新のタイミング
BoussinesqBuoyancy モデルにて、外力を計算した直後に ctx.v += F / (2.0 * rho) と速度を直接更新しています。
問題点: LBMにおいて外力を考慮する場合、**「平衡分布関数の計算に使う速度」と「マクロな物理量として出力する速度」**は厳密に区別する必要があります（Guoの外力モデルなど）。
リスク: 毎ステップこの「半分の力による修正」を直接マクロ変数に上書きし続けると、エネルギー保存則に微小なズレが生じ、長時間のシミュレーションで温度と流速の相関精度が低下します。
